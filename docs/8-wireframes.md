## 개요

### 문서 목적

본 문서는 Python Judge 플랫폼의 모든 주요 화면에 대한 와이어프레임과 UI/UX 설계를 제시합니다. 각 화면의 레이아웃, 주요 컴포넌트, 인터랙션 패턴, 사용자 플로우를 상세히 기술하여 실제 구현 시 참고할 수 있도록 합니다.

### 설계 원칙

| 원칙               | 설명                                                   |
| ------------------ | ------------------------------------------------------ |
| **직관적 UI**      | 학습 곡선 최소화, 명확한 레이블과 아이콘               |
| **빠른 응답**      | 로딩 스피너, 낙관적 UI 업데이트, 즉각적 피드백         |
| **일관성**         | Tailwind CSS 기반 통일된 디자인 시스템                 |
| **피드백**         | 모든 액션에 시각적 피드백 (토스트, 모달, 상태 변화)    |
| **효율성**         | 최소 클릭으로 목표 달성, 키보드 단축키 지원            |
| **안전성**         | 중요 작업 전 확인 모달, 명확한 경고 메시지             |
| **성능 최적화**    | 지연 로딩, 페이지네이션, 적절한 캐싱                   |
| **접근성 우선**    | WCAG 2.1 AA 준수, 키보드 내비게이션, 스크린 리더 지원 |

### 타겟 디바이스

- **우선순위 1**: 데스크탑 (1920x1080, 1366x768)
- **우선순위 2**: 태블릿 (768x1024)
- **우선순위 3**: 모바일 (조회 기능만 제공)

#### 주요 UI 컴포넌트

| 컴포넌트        | 속성                                  |
| --------------- | ------------------------------------- |
| 탭 스위치       | 학생/관리자 전환, 활성 탭 하단 밑줄   |
| 입력 필드       | 테두리 스타일, focus 시 Primary 색상  |
| 체크박스        | "로그인 상태 유지" 기능               |
| Primary 버튼    | "로그인 하기" - 전체 너비             |
| 텍스트 링크     | "비밀번호 찾기", "회원가입"           |
| 모달            | 회원가입 폼 (중앙 오버레이)           |
| 유효성 검증     | 실시간 입력 검증, 에러 메시지 표시    |

#### 인터랙션

1. **탭 전환**: 학생/관리자 클릭 시 폼 초기화
2. **로그인 버튼**:
   - 입력값 검증 후 API 호출
   - 로딩 스피너 표시
   - 성공 시 토큰 저장 후 문제 목록 화면으로 이동
   - 실패 시 에러 토스트 표시
3. **회원가입 링크**: 모달 열기
4. **비밀번호 찾기**: 별도 모달 (군번, 이름 입력)
5. **Enter 키**: 로그인 버튼 트리거

#### 접근성

- 모든 입력 필드에 label 연결 (aria-label)
- 에러 메시지는 aria-live="polite"
- 탭 순서: ID → 비밀번호 → 체크박스 → 로그인 버튼
- 포커스 가시성: focus ring 표시

4. **난이도 필터**: 드롭다운 선택 시 즉시 필터링
5. **hover 효과**: 카드에 마우스 올리면 그림자 강화

```
              검색 입력 → 검색 결과 표시
```

   - 순위 상승: 행 배경색 초록색으로 1초간 깜빡임

   - 아이콘 변화 애니메이션
3. **세션 선택**: 드롭다운 변경 시 해당 세션 스코어보드 로드
4. **수동 새로고침**: 버튼 클릭 시 즉시 API 호출

## 반응형 고려사항

### 데스크탑 (1920px 이상)

- **문제 상세**: 2컬럼 레이아웃 (50% : 50%)
- **사이드바**: 고정 너비 256px
- **컨테이너**: max-w-7xl (1280px)

### 태블릿 (768px - 1919px)

- **문제 상세**: 2컬럼 유지, 비율 조정 (40% : 60%)
- **사이드바**: 접을 수 있는 햄버거 메뉴
- **카드 그리드**: 2열 레이아웃

### 모바일 (768px 미만)

- **문제 상세**: 1컬럼 (세로 스택)
  - 문제 설명 상단
  - 코드 에디터 하단 (전체 화면 모드 옵션)
- **사이드바**: 하단 네비게이션 바로 변경
- **테이블**: 카드 형태로 변환
- **제한 사항**: 관리자 기능은 모바일 미지원 (데스크탑 사용 권장)

### 중단점 (Breakpoints)

```css
sm: 640px
md: 768px
lg: 1024px
xl: 1280px
2xl: 1536px
```

### 반응형 전략

- **Mobile First**: 기본 스타일은 모바일 우선
- **Progressive Enhancement**: 큰 화면에서 기능 추가
- **Fluid Typography**: clamp() 사용하여 유연한 폰트 크기
- **Flexible Grid**: CSS Grid와 Flexbox 혼용

- **ARIA Live Regions**:
  - 채점 결과: aria-live="assertive"
  - 스코어보드 업데이트: aria-live="polite"
  - 에러 메시지: role="alert"
- **대체 텍스트**: 모든 아이콘에 aria-label
- **Skip Link**: "본문으로 건너뛰기" 링크 (화면 최상단)

#### 폼 접근성

- **Label 연결**: 모든 입력 필드에 명시적 label
- **에러 표시**:
  - aria-invalid="true"
  - aria-describedby로 에러 메시지 연결
- **필수 필드**: aria-required="true"
- **도움말 텍스트**: aria-describedby

#### 모달 접근성

- **포커스 트랩**: 모달 열릴 때 포커스 모달 내부로 제한
- **닫기**: Esc 키로 닫기 지원
- **초기 포커스**: 모달 내 첫 번째 입력 필드 또는 닫기 버튼
